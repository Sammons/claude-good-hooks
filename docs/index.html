Get Started in Minutes

Follow these simple steps to start using Claude Good Hooks in your development workflow.

1

Install the CLI

Install Claude Good Hooks globally using npm.

npm install -g @sammons/claude-good-hooks
2

Install a Hook

Install your first hook, like the dirty hook for git status.

npm install -g @sammons/dirty-good-claude-hook
3

Apply the Hook

Apply the hook to your project or globally.

claude-good-hooks apply --global dirty
4

Start Using Claude

Launch Claude Code and see your hooks in action!

claude

API Documentation

Comprehensive guide to creating and using Claude Good Hooks.

CLI Commands

claude-good-hooks list-hooks

List all available hooks

claude-good-hooks apply <hook-name>

Apply a hook with optional arguments

claude-good-hooks remote --add <module>

Add a remote hook source

claude-good-hooks doctor

Check system configuration

Hook Plugin Interface

interface HookPlugin {
  name: string;
  description: string;
  version: string;
  hooks: HookConfiguration;
  customArgs?: CustomArgs;
  makeHook?: (args: any) => HookConfiguration;
}

Hook Events

PreToolUse

Before Claude executes a tool

PostToolUse

After Claude executes a tool

UserPromptSubmit

When user submits a prompt

SessionStart

When Claude session starts

Configuration Options

--global

Apply to ~/.claude/settings.json

--project

Apply to .claude/settings.json

--local

Apply to .claude/settings.local.json

[args...]

Hook-specific arguments

Real-World Examples

See how Claude Good Hooks can enhance your development workflow with these practical examples.

Git Status Hook

Built-in

Automatically show git status and uncommitted changes to Claude before each response. 

# Install the dirty hook
npm install -g @sammons/dirty-good-claude-hook

# Apply with options
claude-good-hooks apply --global dirty --staged --filenames
Result: Claude will always know about your uncommitted changes and can help you manage them.

Custom Linting Hook

Community

Run ESLint or Prettier automatically before Claude modifies any files. 

const lintHook: HookPlugin = {
  name: 'lint',
  description: 'Run linting before file modifications',
  version: '1.0.0',
  hooks: {
    PreToolUse: [{
      matcher: 'Write|Edit',
      hooks: [{
        type: 'command',
        command: 'npm run lint'
      }]
    }]
  }
};
Result: Ensures code quality by running linters before any file changes.

Project Context Hook

Custom

Inject project-specific context like current branch, recent commits, and active issues. 

const contextHook: HookPlugin = {
  name: 'project-context',
  description: 'Add project context to Claude sessions',
  version: '1.0.0',
  hooks: {
    SessionStart: [{
      hooks: [{
        type: 'command',
        command: 'git branch --show-current && git log --oneline -5'
      }]
    }]
  }
};
Result: Claude starts each session with knowledge of your current git context.

Testing Hook

Workflow

Automatically run tests after code changes and report results to Claude. 

const testHook: HookPlugin = {
  name: 'auto-test',
  description: 'Run tests after code changes',
  version: '1.0.0',
  hooks: {
    PostToolUse: [{
      matcher: 'Write|Edit',
      hooks: [{
        type: 'command',
        command: 'npm test -- --reporter=json'
      }]
    }]
  }
};
Result: Immediate feedback on test results after any code modification.
